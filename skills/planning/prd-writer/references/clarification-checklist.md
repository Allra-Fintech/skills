# 명확화 질문 체크리스트

이 문서는 `prd-writer`가 정보가 모호할 때 어떤 질문을 먼저 해야 하는지 정리한 체크리스트다.

## 질문을 먼저 해야 하는 대표 상황

- 문제는 있는데 누가 대상 사용자인지 불명확하다.
- 기능 범위와 제외 범위가 섞여 있다.
- 성공 기준이 전혀 없다.
- 정책, 보안, 운영 제약이 있는지 알 수 없다.
- 기존 기능 개선인지 신규 기능인지 모호하다.
- 참고해야 할 문서가 있다고 했지만 실제 내용이 없다.
- API 계약, 화면 설계, 운영 절차 중 무엇이 기준인지 불분명하다.

## 우선순위 높은 질문

### 1. 문제와 대상 사용자

- 이번 기능이 해결하려는 가장 큰 문제는 무엇인가?
- 1차 대상 사용자는 누구인가?
- 사용자군이 여러 개라면 이번 PRD의 중심 사용자는 누구인가?

### 2. 범위와 제외 범위

- 이번 문서에 포함되는 범위는 어디까지인가?
- 이번 단계에서 명시적으로 제외할 항목은 무엇인가?
- 기존 기능 개선이라면 어떤 화면 또는 기능을 기준으로 보는가?

### 3. 성공 기준

- 이 기능이 성공했다고 판단하는 기준은 무엇인가?
- 수치 목표가 없다면 최소한 어떤 행동 변화를 기대하는가?

### 4. 제약 사항

- 정책, 보안, 운영상 반드시 지켜야 할 제약이 있는가?
- 일정이나 우선순위상 이번 단계에서 포기해야 하는 것은 무엇인가?

### 5. 참고 자료

- 회의 메모, 정책 문서, API 문서, 화면 설계 중 반드시 참고해야 할 자료가 있는가?
- 기존 기능 링크나 캡처가 있는가?

## 질문 방식 규칙

- 추상 질문보다 문서 작성에 직접 필요한 질문을 한다.
- 한 번에 너무 많은 질문을 던지지 않는다.
- 답이 없으면 문서를 멈추기보다, 영향이 큰 항목만 추가 질문하고 나머지는 `가정`과 `오픈 이슈`로 분리한다.

## 질문 후 처리 규칙

- 답을 받으면 즉시 PRD 구조에 반영한다.
- 답을 받지 못한 항목은 본문 확정 사실처럼 쓰지 않는다.
- 영향이 큰 미확정 항목은 `오픈 이슈`와 `다음 액션`에 남긴다.
