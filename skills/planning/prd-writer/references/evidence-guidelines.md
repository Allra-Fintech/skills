# 출처 / 근거 / 가정 가이드

이 문서는 `prd-writer`가 어떤 내용을 근거로 쓰고, 어떤 내용은 가정으로 분리해야 하는지 판단할 때 참고한다.

## 무엇을 근거로 볼 것인가

아래 항목은 `참고 자료 / 근거`에 들어갈 수 있다.

- 회의 메모
- 슬랙 또는 메신저 논의 기록
- 기존 기능 문서
- 정책 문서
- 운영 가이드
- API 계약 또는 스펙 문서
- VOC
- 데이터 리포트
- 와이어프레임
- 화면 캡처

## 근거로 적으면 안 되는 것

- 입력으로 주어지지 않은 링크
- 존재 여부가 불명확한 정책 문서
- 확인되지 않은 수치
- 일반적으로 그럴 것이라는 추정

## 출처와 가정 구분 기준

### 출처 / 근거로 적는다

- 사용자 입력에 직접 포함된 내용
- 명시적으로 참고하라고 주어진 자료
- 기존 기능 또는 정책에서 확인 가능한 내용
- 데이터나 VOC처럼 확인 가능한 정보

### 가정으로 적는다

- 명시적 근거는 없지만 문서 진행을 위해 잠정 판단한 내용
- 답변을 기다리는 동안 임시로 둔 범위 설정
- 대상 사용자, 운영 방식, 측정 방식에 대한 추론

## 근거 부족 시 처리 규칙

- 사실처럼 쓰지 않는다.
- 먼저 질문할 수 있으면 질문한다.
- 답을 받을 수 없으면 `가정`과 `오픈 이슈`로 분리한다.
- 영향이 큰 가정은 `다음 액션`에도 연결한다.

## 좋은 예시

### 좋은 예시

- `운영자 승인 단계가 병목으로 작동하고 있다. (Ref-2)`
- `결제 성공 후 5분 내 입금 확인이 중요하다는 VOC가 반복적으로 확인되었다. (Ref-3)`

### 나쁜 예시

- `대부분의 사용자는 바로 알림을 원할 것이다.`
- `보안 정책상 반드시 2차 승인이 필요하다.`

위 문장들은 근거가 없으면 사실이 아니라 가정이다.

## 근거 연결 규칙

- 요구사항, 지표, 시나리오, 리스크 중 근거가 분명한 항목은 가능한 범위에서 `Ref-*`를 붙인다.
- 모든 문장에 억지로 근거를 붙일 필요는 없지만, 핵심 주장에는 근거나 가정이 보여야 한다.
